# 하루 시간 블록 (하드 모드 10h+)

번아웃 방지가 성공의 절반. 시간 블록 고정 → 의사결정 피로 감소.

## 표준 일과

| 시간 | 블록 | 용도 |
|---|---|---|
| 08:30-09:00 | 웜업 (30m) | 어제 `daily-log.md` 읽기 + 오늘 커리큘럼 훑기 |
| 09:00-11:00 | 읽기 (2h) | 공식 문서/튜토리얼 read-only. 코드 짜지 말 것 |
| 11:00-12:30 | 튜토리얼 따라치기 (1.5h) | notebook 실행만. 변형 X |
| 12:30-13:30 | 점심 + 산책 (1h) | 화면 끄기 |
| 13:30-16:30 | 본 실습 (3h) | 프로젝트 디렉토리에서 **변형 과제** 작성 |
| 16:30-17:00 | 쉬는 시간 (30m) | 진짜 쉬기. 트위터 X |
| 17:00-19:00 | 디버깅 + 확장 (2h) | 막힌 거 해결 + stretch 과제 |
| 19:00-20:00 | 저녁 (1h) | — |
| 20:00-21:00 | 키워드/개념 정리 (1h) | `notes/keywords.md`, `notes/concepts.md` |
| 21:00-21:30 | 하루 회고 (30m) | `notes/daily-log.md` + git commit |

**총 9.5h 공부 + 2h 휴식**. 10h 목표 무난히 달성.

## 요일별 변형

### 월/수/금 — 빡센 날
위 스케줄 그대로.

### 화/목 — 호흡 고르기
- 오전 2h 읽기 대신 어제의 복습/디버깅
- 오후 3h 그대로
- 저녁 정리 대신 cheatsheet 만들기
- **총 8h 정도** — 의도된 감속

### 토/일 — 실전 몰입
- 오전 3-4h 한 번에 큰 실습
- 오후 3-4h 다른 실습
- 저녁은 짧게 회고

## 블록 규칙

1. 한 블록이 **끝나기 전에 블록 갈아타지 않는다**. 막히면 다음 블록에서.
2. Pomodoro 25/5 을 블록 안에서 사용.
3. **Read-only 시간에 키보드 손 대지 말 것** — 이게 안 되면 얕게 스치기만 하게 됨.
4. 디버깅이 30분 넘으면 에러 그대로 Claude한테 붙여넣어라. 자존심 낭비 금지.
5. 매 블록 시작 시 1문장 "이 블록에서 뭘 끝낼 것인지" 쓰기.

## 휴식 원칙
- 점심 전후 **완전히 화면 밖**
- 저녁 식사 동안 **노트북 닫기**
- 21:30 이후는 **공부 금지** (수면 우선)

## 안티패턴
- ❌ 새벽 2시까지 뭔가 돌아가게 만들기 (다음날 4시간 손해)
- ❌ 읽기 → 실습을 반복해서 섞기 (컨텍스트 스위칭 오버헤드)
- ❌ 한 문제에 2시간 이상 (질문 없이)
- ❌ Slack/Twitter/유튜브 "검색" (전부 2시간 사라짐)

## 컨디션 관리
- 카페인 오전만. 오후는 물/녹차
- 책상 앞 6시간 넘게 앉아있지 말 것 → 서서 일하기 또는 산책
- 눈 깜빡임 의식. 20-20-20 규칙 (20분마다 20ft 거리 20초 쳐다보기)
