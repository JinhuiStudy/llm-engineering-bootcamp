# 개념 정리 노트

학습 중 직접 깨달은 것/헷갈렸던 것/모델 별 차이를 정리한다. 초기에는 비어있음 — 매일 채워나가기.

## 템플릿

각 개념은 이 포맷으로:

```
## <개념>
- 한 줄 요약:
- 언제 쓰는가:
- 자주 하는 실수:
- 코드 예시 또는 링크:
```

---

## (아직 비어 있음 — 학습하면서 채울 것)

### 예시: Structured Output
- 한 줄 요약: JSON Schema를 주고 LLM이 100% 스키마대로 출력하게 만드는 것.
- 언제 쓰는가: 파싱 실패가 치명적일 때 (DB 저장, 후속 tool 호출).
- 자주 하는 실수: `required` 누락, enum 없이 자유 문자열 허용.
- 코드 예시: `projects/day03-structured-extractor/schemas.py`

### 예시: Chunk overlap이 retrieval에 미치는 영향
- 한 줄 요약: overlap 0이면 경계에 걸친 답을 놓침, 너무 크면 중복.
- 언제 쓰는가: ingest 파이프라인 튜닝.
- 자주 하는 실수: 경계 문장 자름.
- 관찰: Day 6 실험에서 overlap 50 vs 200 차이 기록
